# Micro-Interaction

**Micro-interactions** are single-purpose events found throughout a product. They are the key details that make a product feel "alive" and "human".

## Core Functions

1.  **Feedback Status**: Confirming that an action has occurred (e.g., button click ripple effect).
2.  **System Status**: Download progress bars, loading spinners.
3.  **Guiding Users**: Highlighting interactive elements.
4.  **Preventing Errors**: Password strength indicators, vibration feedback.

## Design Principles

*   **Fast**: Micro-interactions should complete within 0.1s - 0.4s.
*   **Subtle**: Don't be overwhelming; they should be natural supplements.
*   **Consistent**: Maintain the same interaction logic throughout the application.

```css
/* Simple Button Micro-interaction */
.btn {
  transition: transform 0.1s;
}
.btn:active {
  transform: scale(0.95);
}
```
