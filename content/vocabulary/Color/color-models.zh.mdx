# Color Models (色彩模型)

色彩模型定义了如何在计算机中表示颜色。理解不同的模型有助于开发者根据场景选择最合适的表达方式。

## 核心模型

### 1. RGB (Red, Green, Blue)
*   **原理**: 基于光的加色混合。
*   **适用**: 硬件显示（屏幕）。
*   **缺点**: 不直观，很难通过调整数值来让颜色"更亮一点"或"更鲜艳一点"。

### 2. HSL (Hue, Saturation, Lightness)
*   **原理**: 基于人类对颜色的感知。
    *   **色相 (Hue)**: 颜色的种类（0-360度）。
    *   **饱和度 (Saturation)**: 颜色的纯度。
    *   **亮度 (Lightness)**: 颜色的明暗。
*   **适用**: 设计系统、动态调整颜色。

### 3. LCH / OKLCH (Lightness, Chroma, Hue)
*   **原理**: 感知均匀的色彩空间。
*   **优点**: 解决了 HSL 在不同色相下视觉亮度不一致的问题（例如，HSL 中纯黄色比纯蓝色看起来亮得多，但在 OKLCH 中它们的亮度值会更能反映人眼感知）。
*   **适用**: 现代 Web 设计，无障碍设计。

## 比较

| 模型 | 直观性 | 感知均匀性 | 浏览器支持 |
| :--- | :--- | :--- | :--- |
| **RGB** | 低 | 低 | 极好 |
| **HSL** | 高 | 中 | 极好 |
| **OKLCH**| 高 | 高 | 良好 (现代浏览器) |
