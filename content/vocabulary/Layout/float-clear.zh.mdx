# 漂浮的盒子

想象一下报纸上的排版：一张图片位于左侧，文字紧密地围绕在图片右侧和下方。这就是 `float` 最初诞生的目的。

## 核心概念

### 1. Float (浮动)
当一个元素设置了 `float`，它会尽可能向左或向右移动，直到触碰到容器边缘或另一个浮动元素。
- `float: left`：向左浮动。
- `float: right`：向右浮动。
- `float: none`：默认值，不浮动。

**关键行为**：浮动元素会**脱离文档流**（半脱离），文字内容会意识到它的存在并围绕它，但块级父容器会“看不见”它，导致高度塌陷。

### 2. Clear (清除)
`clear` 属性指定一个元素是否允许其旁边有浮动元素。
- `clear: left`：左侧不允许有浮动元素（换行）。
- `clear: right`：右侧不允许有浮动元素。
- `clear: both`：两侧都不允许（最常用）。

## 经典问题：高度塌陷

如果父容器只包含浮动子元素，父容器的高度会变成 0。因为浮动子元素“漂”起来了，撑不开父容器。

### 解决方案：Clearfix (清除浮动)

这是前端开发中最经典的 Hack 技巧，用于修复高度塌陷：

```css
/* 现代 Clearfix 写法 */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}
```

## 何时使用？

- **✅ 推荐**：实现图文混排、文字环绕效果。
- **❌ 不推荐**：用于构建整个页面的栅格布局（请使用 Flexbox 或 Grid）。
