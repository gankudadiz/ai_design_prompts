# 尺子的刻度

在现实世界中，我们用米或英寸测量距离。在 CSS 世界里，我们有更多的选择，而选择哪种“尺子”决定了你的布局是否灵活。

## 1. 绝对单位：px (像素)
- **定义**：屏幕上的一个点（逻辑像素）。
- **特点**：固定不变。设置 `200px`，它永远是 `200px`。
- **场景**：边框宽度、由于设计限制必须固定的尺寸、最小尺寸 (`min-width`)。

## 2. 相对字体单位：em 和 rem
这些单位基于字体大小，让布局随文字缩放。

- **em**：相对于**父元素**的字体大小。
  - 陷阱：如果多层嵌套，会产生乘法效应（1.2 * 1.2 * 1.2...），容易失控。
- **rem (Root em)**：相对于**根元素 (html)** 的字体大小。
  - **推荐**：大多数现代响应式排版都使用 `rem`。如果用户调整了浏览器默认字号，整个页面会等比缩放。

```css
html { font-size: 16px; }
.box {
  width: 10rem; /* = 160px */
  padding: 1rem; /* = 16px */
}
```

## 3. 相对视口单位：vw, vh
这些单位基于**浏览器窗口 (Viewport)** 的尺寸。

- **vw (Viewport Width)**：视口宽度的 1%。`100vw` = 满屏宽度。
- **vh (Viewport Height)**：视口高度的 1%。`100vh` = 满屏高度。
- **场景**：全屏 Hero Section (`height: 100vh`)、响应式大标题。

## 4. 相对父容器单位：%
- **定义**：相对于**直接父元素**的尺寸。
- **特点**：`width: 50%` 意味着占据父容器宽度的一半。
- **注意**：`height: 50%` 通常需要父容器有明确的高度才生效。

## 最佳实践指南

| 场景 | 推荐单位 | 理由 |
| :--- | :--- | :--- |
| **整体布局宽度** | `%`, `max-width` | 适应不同屏幕 |
| **间距 (Padding/Margin)** | `rem` | 随基础字号呼吸 |
| **文字大小** | `rem` | 尊重用户浏览器设置 |
| **边框** | `px` | 需要精细控制 |
| **全屏背景** | `vh`, `vw` | 填满视口 |
