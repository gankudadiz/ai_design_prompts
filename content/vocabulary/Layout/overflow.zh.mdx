# 盒子装不下了怎么办？

当你买了一个小箱子，却试图塞进一件大棉袄时，会有部分棉袄露在外面。在网页布局中，这就是 **Overflow (溢出)**。`overflow` 属性决定了如何处理这些“露出来”的内容。

## 核心属性值

### 1. visible (默认值)
**“随它去吧”**。
- 内容会溢出容器框外，覆盖在其他元素上方。
- 这是浏览器的默认行为，为了确保内容总是可见的。

### 2. hidden
**“眼不见为净”**。
- 超出容器的部分会被直接**裁剪**（切掉）。
- 用户无法通过滚动看到被切掉的内容。
- 常用于：制作圆形头像（配合 `border-radius`）、清除浮动、隐藏装饰性元素的边缘。

### 3. scroll
**“强制滚动条”**。
- 无论内容是否溢出，总是显示滚动条。
- 很少使用，因为通常我们只希望在需要时才显示滚动条。

### 4. auto (推荐)
**“智能滚动”**。
- 如果内容溢出，显示滚动条；如果没溢出，不显示。
- 这是实现“局部滚动区域”（如侧边栏、聊天窗口）的标准方式。

```css
.sidebar {
  height: 100vh;
  overflow-y: auto; /* 仅在垂直方向需要时滚动 */
}
```

## 文本溢出处理 (Text Overflow)

对于单行文本，我们经常需要“超出部分显示省略号”的效果。这需要三个属性配合：

```css
.truncate {
  white-space: nowrap;      /* 1. 强制不换行 */
  overflow: hidden;         /* 2. 隐藏溢出内容 */
  text-overflow: ellipsis;  /* 3. 用省略号(...)代替被修剪的文本 */
}
```
