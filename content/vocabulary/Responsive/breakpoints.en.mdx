# Breakpoints

**Breakpoints** are specific pixel width values in responsive design. When the viewport width reaches these values, the webpage layout changes to adapt to different screen sizes.

## Common Breakpoint Systems (Tailwind CSS Example)

Modern frameworks typically use a set of predefined breakpoints:

*   **sm (Small)**: `640px` - Large phones / Landscape phones
*   **md (Medium)**: `768px` - Tablets (iPad)
*   **lg (Large)**: `1024px` - Small laptops / Landscape tablets
*   **xl (Extra Large)**: `1280px` - Standard desktop monitors
*   **2xl**: `1536px` - Large monitors

## How to Choose Breakpoints?

You shouldn't design breakpoints for *specific devices* (like iPhone 14 Pro), but rather for *content* or *common size ranges*.

1.  **Content First**: When your layout looks "broken" or too crowded at a certain width, add a breakpoint there.
2.  **Mobile First**: Follow the mobile-first principle; handle small screens first, then expand to larger screens using `min-width` breakpoints.

## Code Example

```css
/* Base styles (Mobile) */
.card {
  width: 100%;
}

/* When screen width is greater than 768px (md) */
@media (min-width: 768px) {
  .card {
    width: 50%; /* Becomes 2 columns */
  }
}

/* When screen width is greater than 1024px (lg) */
@media (min-width: 1024px) {
  .card {
    width: 33.33%; /* Becomes 3 columns */
  }
}
```
