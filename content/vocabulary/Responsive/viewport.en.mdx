# Viewport

The **Viewport** is the user's visible area of a web page. On desktops, it is typically equal to the browser window size; however, on mobile devices, it is more complex.

## Why Do We Need the Viewport Meta Tag?

In the early days of smartphones, to display wide webpages designed for desktops on small screens, mobile browsers would default to a wide viewport width (usually 980px) and then shrink the page to fit the screen. This required users to zoom in to read text.

To tell the browser "This is a responsive site, please use the device's actual width," we need to use the `<meta name="viewport">` tag.

## Standard Configuration

This is essential code for all responsive websites, usually placed in the `<head>`:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### Parameters Explained

*   **width=device-width**: Sets the viewport width to the device's physical width (e.g., iPhone 12 is 390px, instead of the default 980px).
*   **initial-scale=1.0**: Sets the initial zoom level to 1:1 (no zooming).

## Other Parameters (Use with Caution)

*   `maximum-scale`: The maximum zoom level allowed.
*   `user-scalable=no`: **Strongly discouraged**. Disabling user zooming severely harms accessibility, preventing visually impaired users from reading content.
