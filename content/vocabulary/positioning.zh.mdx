# 元素去哪儿了？

默认情况下，HTML 元素像排队一样一个接一个地出现（文档流）。但有时你想把一个元素“钉”在屏幕角落，或者让它覆盖在其他元素上面，这就需要用到 `position`。

## 五种模式的通俗解释

### 1. static (默认值)
**“听话排队”**。
元素按照正常的顺序排列，你不能随意移动它。`top` / `left` 对它无效。

### 2. relative (相对定位)
**“灵魂出窍”**。
元素还在原来的位置占个坑（保留空间），但它的“灵魂”（视觉效果）可以稍微偏离一点。
*   **用途**：通常用来微调位置，或者作为 `absolute` 子元素的参考系（父级）。

### 3. absolute (绝对定位)
**“断绝关系”**。
元素完全脱离了原来的队伍，不再占坑。它会寻找最近的一个“非 static”的长辈元素作为参考点进行定位。
*   **用途**：制作下拉菜单、角标、覆盖层。

### 4. fixed (固定定位)
**“粘在屏幕上”**。
不管页面怎么滚动，它都死死地钉在浏览器窗口的某个位置。
*   **用途**：回到顶部按钮、固定导航栏。

### 5. sticky (粘性定位)
**“见风使舵”**。
它在 `relative` 和 `fixed` 之间切换。平时它跟着页面滚动，但当滚到屏幕边缘时，它就会“粘”在那里不动。
*   **用途**：通讯录的字母标题、表格表头。
